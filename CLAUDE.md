# jfighter

Hobby top-down space roguelite built with LibGDX (Java, Gradle). Run with `./gradlew lwjgl3:run`; compile check with `./gradlew :core:compileJava`. Each run (`new GameState()` from the title screen) gets a random sector map and random crew (names/stats from pools in GameState); the ship layout is fixed. Reaching the final map node ("END") shows a victory modal returning to the title screen.

All game code lives in `core/src/main/java/be/jfighter/` (flat package). `lwjgl3/` is only the desktop launcher.

## Structure

- `JFighter` — the `Game` subclass; screens are swapped via `game.setScreen(...)`. Also owns display settings: launches windowed 1440x810 by default (`WINDOW_WIDTH/HEIGHT`), F11 toggles fullscreen from any screen (polled in `render()`), and the chosen mode persists via LibGDX `Preferences` ("jfighter") applied in `create()` (`setWindowSize`/`setFullscreen`).
- `OptionsScreen` — display options reachable from the title screen: windowed presets (`JFighter.WINDOW_SIZES`) + fullscreen, current mode highlighted green; ESC/BACK returns to the title.
- `GameState` — mutable session state passed to every screen (holds `OverworldMap` and `credits`).
- `OverworldScreen` + `OverworldMap` + `Node` — split screen. Top half: `ShipDeckView`, a Nox-style live feed of the carrier deck — flat deck-coordinate simulation rendered through a steep oblique projection (`px`/`py` are the only projection points; `SCALE`/`SQUASH`/`Z_FACTOR`). Rooms flank a spine corridor (deck-space `ROOMS`, same indices as `ROOM_NAMES`: engine, cargo, hangar, quarters, weapons, bridge, medical bay, life support); the `AIRLOCKS` array defines spur+chamber airlocks (A south, B north aft of the bridge; chambers are not stationable rooms). All doors are clickable (0..7 = room corridor doors, then inner/outer per airlock via `innerDoor(a)`/`outerDoor(a)`): a click toggles held-open (`GameState.doorHeldOpen`, bright white-green with latch posts); toggled off = automatic (opens for crew within `DOOR_CREW_RADIUS`, closes behind; outer doors never auto-open). Oxygen sim (`GameState.oxygen`, compartments = rooms + corridor(+spurs) + one per chamber): open doors equalise toward the volume-weighted mix (`OXY_FLOW`), each open outer door independently vents its chamber (`VENT_FLOW`) so multi-airlock bleed stacks; life support regenerates all compartments at `OXY_REGEN` (hook for crew stats later); floors redden with oxygen deficit, ship-wide O2 % shown top-right under the fuel readout (starting fuel: 8). Crew have `hp` (`CrewMember.MAX_HP`): compartments below `SUFFOCATE_THRESHOLD` oxygen drain `SUFFOCATE_DPS`, the medbay heals `MEDBAY_HEAL_PER_SEC` while inside; taking damage sets `CrewMember.damageFlash`, blinking the avatar and the roster health bar red (`damageBlinkOn`); at 0 hp the figure keels over 90° (`Sim.fallT`, `bodyX/bodyY` rotate all body points around the feet), stops moving/lighting rooms, and shows KIA in the roster. Roster rows show an hp bar under each name; deck figures have no initial nametag. No revival — death is permanent for the run; cutaway walls (full back-wall faces, stub door posts on corridor edges), darkness with flickering light pools (blending enabled inside `renderShapes`), rooms brighten when occupied, scanline + CAM/REC feed overlay. Crew are stick figures (swap `drawFigure` for sprites) that physically walk to stations via corridor waypoints (`planPath`); positions persist in `CrewMember.deckX/deckY`. Crew roster panel top-left (`ROSTER_*` in OverworldScreen): `GameState.crew` holds `CrewMember`s, each with a primary skill (+2) and a different secondary skill (+1) from the `Skill` enum (`bonusFor`). Each room except quarters has one station governed by a skill (`ShipDeckView.ROOM_SKILL`): the earliest-assigned living crew in a room mans it (`stationHolder`, FIFO via `assignedAt` stamped from `GameState.nextStationSeq()` on stationing) and stands at the console; others queue mid-room, and moving/losing the holder auto-promotes the next in line (idle re-path check in update). `roomStat(room)` = +1 flat for a manned station plus the holder's skill bonus, once physically at the post (so +1..+3); shown green on the room label ("ENGINE ROOM +3") — no gameplay effect yet. `station` = room index, -1 = off duty (idles in quarters). Click a roster row to select, click a compartment to station — the crew member walks there. Bottom half (below `MAP_TOP` = 270): procedurally generated sector map — `OverworldMap.generate()` lays jittered columns from HOME (node 0) to the final node (`lastNodeId`, orange objective ring); every column node gets a forward and a backward edge so start→end connectivity is guaranteed by construction; middle nodes get ≥1 TRADER and ≥1 LOOT. Cosmetic `Decal`s (scan rings/crosses/hazards) + faint grid dress the map. Traveling to another node costs `TRAVEL_FUEL_COST` (1) fuel via `travelTo` — at 0 fuel you're stuck (map shows OUT OF FUEL). Clicking a reachable non-HOME node opens an encounter dialog voiced by the ship's AI addressing the Captain (`encounterPrompt/Option1/Option2` per node type): option 1 = travel + enter the instance, option 2 = travel but skip the encounter, click outside = dismiss without traveling. Clicking HOME travels directly; END triggers victory. The map title is a random constellation sector name per run (`OverworldMap.sectorName`). `OverworldMap.isReachable` only allows current node + direct neighbors.
- `GameScreen` — combat instance. SPACE fires `Projectile` capsules. Ship physics live in `Player`: exponential drag with a min-drift floor (same values as cargo), so each throttle step maps to a terminal velocity.
- `LootScreen` — looting minigame in a 1440x810 arena (bigger than the screen; `FitViewport` scales it down; HUD uses a separate 960x540 ortho matrix). The net is a rope simulation: chains of `NetPoint`s with velocities + drag, solved by max-distance constraints (`NET_REST_LEN`, `NET_ITERATIONS` Gauss-Seidel passes — long chains are deliberately stretchy). SPACE deploys a net that pays out from the ship's tail (anchored there; capped at `NET_MAX_LENGTH` = 50 ship lengths); SPACE again *cuts* it into a free-floating net. Net points touching cargo stick to its surface (`stickNetsToCargo`) and rope tension tows the crate (`CARGO_TOW_FACTOR/IMPULSE`); casting over a loose strand auto-cuts the payline and splices it on (`connectCastNet`); otherwise free-net contact behaviour depends on what touches (`interactNets`): two open strands splice end-to-end (`spliceNets`, joint slides to the nearest end of each and the rope constraints cinch it), an open strand ties its floating ends onto a sealed ring (`tieStrandToRing`, `NetLink` cinched to `NET_LINK_REST`; a strand tied at both ends that caught nothing disintegrates via `dissolveEmptyCatches`), and two sealed rings pop into one massive ring (`popMerge`: inner links dissolve, points reorder by angle around the merged centroid). The ship flies through nets (no strand collision; only the hulk pushes strands) but still collides with cargo. E fires a tractor hook backward from the tail (`HookState` STOWED→EXTENDING→LATCHED/RETRACTING): latches the first *free-net* point the tip passes (`latchNearestNetPoint`; the deployed tether is excluded) and becomes a rope tow line to the net (`towNet`, stretch-only; pull propagates through the net's constraints to any stuck cargo); E again releases, border wrap snaps it, trimming the hooked strand auto-releases. Free nets whose strand crosses itself (`closeLoops`, crossing points ≥ `LOOP_MIN_SEGMENTS` apart along the rope) seal into a closed ring: tails outside the loop are trimmed (`dropPoint` dissolves their tangle links), the ring gains a last↔first constraint, relaxes toward a true circle while cinching like a drawstring (`roundClosedLoops`: `ringRadius` shrinks at `RING_SHRINK_RATE` down to `minRadius` = `RING_MIN_FRACTION` of the sealing radius), and draws brighter orange with a faint criss-cross fishnet mesh over the interior (`drawFishnetMesh`). Sealing (`sealRing`) snapshots the crates inside the ring polygon as the net's `contents` (`captureContents`, `Intersector.isPointInPolygon`): a sealed ring only sticks to its own catch and ignores the hulk (open strands still get shoved by it); a loop that seals empty — no contents and nothing stuck — disintegrates into drifting sparks (`disintegrate`/`Spark`). Scoring: drag cargo into the dashed EXIT circle at the insertion point (`EXIT_*`); no instant loop-catch anymore. Cargo: exponential drag with a min-drift floor, mass-weighted ship bumps, equal-mass cargo-cargo collisions, hulk gravity. Ship border wrap auto-cuts the deployed net. The craft here is a pincer salvage pod (`effects.setPincerHull(true)` → `ShipRenderer.drawPincer`): F traps the nearest crate within `PINCER_RANGE` of the claw centre (up to `PINCER_CAPACITY` = 3, pinned to `PINCER_SLOTS` in the jaws, excluded from all cargo physics and netting); F with nothing in reach releases the clump at ship velocity; delivery at the exit auto-empties the jaws.
- `TraderScreen` — trader instance; F buys fuel with credits.
- `Player` — shared ship physics: stepped throttle (UP/DOWN tap ±10%, `throttle` 0..10, ship burns continuously; `thrustLevel` ramps toward the setting), rotational inertia (keys apply torque, `angularVel` decays), speed clamp, and `wrapAround` with period = world size so wrap ghosts line up seamlessly (in LootScreen a wrap snaps the net). No brake: flip and burn.
- `SpaceEffects` — shared "space feel" for both flight instances, one instance per screen: 3-layer parallax starfield, drifting wireframe debris, RCS puffs on rotate/brake, blinking nav lights (red left/green right), speed-based camera zoom-out, edge-wrap ghost rendering, velocity drift indicator, and fuel-burning flight input (`handleFlightInput`). All tuning constants at the top of the class. HUDs must use a separate ortho matrix so camera zoom doesn't move them.
- `SoundFx` — procedurally synthesised WAVs (thruster loop, catch twang, cargo thud) written to `~/.jfighter/sfx/` on first run; owned by `JFighter` (`game.sfx`), degrades to silence on failure. Screens start/stop the thruster loop in `show()`/`dispose()`.
- `GameState` also tracks `fuel`/`maxFuel`; fuel is purely an overworld travel resource (instances fly free; buy fuel at the trader). The thruster sound loop is currently disabled (SoundFx still has it for later). LootScreen has a central derelict hulk: inverse-square gravity well (capped) pulling ship and cargo, immovable collision circle, it blocks the net, and animated violet rings fall inward around it to signal the pull (`GRAVITY_RING_*`). `ShipRenderer` — static B-2 wireframe drawing shared by GameScreen and LootScreen; caller sets the transform matrix and Line shape type.

## Conventions

- Screens use a `FitViewport` at the shared 16:9 virtual resolution `JFighter.WORLD_WIDTH x WORLD_HEIGHT` (960x540) — except LootScreen, whose arena is 1440x810 — and dispose their own resources in `hide()`. HUDs always render through a 960x540 ortho matrix. The desktop window (Lwjgl3Launcher) is 1440x810.
- Rendering is `ShapeRenderer` wireframes (green ship, yellow loot/net) + `BitmapFont` HUD; no textures.
- ESC returns from any instance to the overworld map.
